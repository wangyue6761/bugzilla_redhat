Common Vulnerabilities and Exposures assigned an identifier CVE-2010-1864 to the following vulnerability:\n\nThe addcslashes function in PHP 5.2 through 5.2.13 and 5.3 through 5.3.2 allows context-dependent attackers to obtain sensitive information (memory contents) by causing a userspace interruption of an internal function, related to the call time pass by reference feature.\n\nReference:\nhttp://php-security.org/2010/05/03/mops-2010-006-php-addcslashes-interruption-information-leak-vulnerability/index.html\n\n

\n\n\n \n Comment 3\n \n\n \n Tomas Hoger\n\n \n\n \n \n \n\n \n 2010-07-23 13:32:18 UTC\n \n\n
\n\n \n\n\n \n \n\n\n\n 
Interruption vulnerabilities allow untrusted script author to bypass restrictions such as safe_mode or open_basedir.  Due to the limitations of the implementation of these mechanisms, they can not be relied on as security features.  Therefore flaws bypass of there restrictions are not considered security sensitive.  For additional details, refer to bug #169857, comment #1.\n\n*** This bug has been marked as a duplicate of bug 169857 ***\n\n
